## HR-01 请假申请 `hr-leave-apply`

**Page Pattern**：`FlowApply`（E-47）  
**开源参照**：JeecgBoot [请假案例](http://www.jeecgboot.com/leave/) · RuoYi [假期余额](https://jishuzhan.net/article/2051826065543331841)

### 区域布局

```
FgTitle「请假申请」
FgForm (form_main)
  ├─ FgSelect leaveType      dict://leave_type
  ├─ FgDate startTime/endTime  datetime
  ├─ FgNumber days             linkage←日期
  ├─ FgTextarea reason
  ├─ FgUserSelector agentUser
  └─ FgUpload attachments      linkage 病假 required
[提交申请] [重置]
  submit → submitSubmission + navigate hr-leave-list
```

### 功能点

| # | 功能 | 规则 | 来源 |
|---|------|------|------|
| F1 | 假别字典 | 必填 | JeecgBoot |
| F2 | 日期+天数联动 | end≥start | 通用 |
| F3 | 病假附件必填 | linkage | 模块 doc |
| F4 | 提交 Webhook 启流 | submission.created | JeecgBoot startProcess |
| F5 | 余额不足提示 | 提交前 API | RuoYi getLeaveSummary |
| F6 | 撤回说明 | 首节点未审可撤回 | JeecgBoot 撤回 |
| F7 | 制度 RAG | Sidebar A-08 | O2OA 员工自助 |

### Flow

部门经理 → HR → 结束（内置模板）

### API

POST `/api/submissions` · Webhook → 请假审批

### 缺口

E-47 FlowApplyPage · S-10 LeaveBalance · A-08 Sidebar

## HR-02 请假台账 `hr-leave-list`

**Page Pattern**：`CrudList`（E-45）  
**开源参照**：JeecgBoot LeaveList + useListPage + searchFormSchema

### 区域布局

```
[A] keyword|leaveType|status|dateFrom|dateTo
[B] 发起申请 | 导出 Excel
[C] 9 列 + flowStatus + 当前节点
[D] 查看(弹窗) | 审批(全屏)
[F] Dialog: Descriptions + FlowTimeline → 全屏审批
```

### 功能点

| # | 功能 | 来源 |
|---|------|------|
| F1 | 9 列含 data.* | JeecgBoot columns |
| F2 | 5 项搜索 | searchFormSchema |
| F3 | 导出 xlsx | useListPage export |
| F4 | 弹窗详情 | Modal/Dialog 模式 |
| F5 | 全屏审批 | navigate detail |
| F6 | 列筛选假别/状态 | filterable |
| F7 | 撤回 | JeecgBoot beforeRetract |

### Editor 组件树（目标 E-45 props）

见 interface-specs/README HR-02 草案；现状 `buildHrLeaveListSchema` Board JSON

### API

GET `/submissions/{applySchemaId}` · GET `/business/hr/leave/detail` · export
