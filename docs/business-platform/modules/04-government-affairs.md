# 模块落地 — 行政审批

> 优先级：**P1–P2** | Phase：**3**

## 1. 模块概述

行政审批模块面向政务场景：事项受理、并联审批、证照管理、政策发布、督查督办。技术特点是 **多部门并行（Flow 并行网关）**、**材料清单**、**时效监控**、**政策 RAG**。

## 3. 核心界面逻辑

### GA-01 事项受理

**服务对象：** 窗口工作人员

**表单结构（Tabs）：**

```
Tab1 申请人信息
  - 个人/企业 Switch
  - 姓名/企业名、证件号、联系方式
  - Cascader 事项类型（字典 gov_case_type）

Tab2 申请材料
  - 材料清单 Table（材料名、是否必需、已上传）
  - Upload 多文件
  - AI 完整性检查按钮 (A-04)

Tab3 办理信息
  - 承诺时限（自动从事项类型带出）
  - 备注
```

**提交逻辑：**

```
1. 创建 submission
2. 启动「并联审批」Flow
3. 生成办件编号 caseNo（Server 序列号扩展）
```

### GA-03 办件详情 — 并联审批

**Flow 结构：**

```
受理 → 并行网关 ─→ 部门A审批 ─┐
              ├── 部门B审批 ─┼→ 汇聚 → 决定 → 结束
              └── 部门C审批 ─┘
```

**界面：**

- Descriptions 办件信息
- 并联进度：各分支状态 Gauge/Steps（扩展 Widget 或 FgFlowTimeline 增强）
- 各部门意见 Tab

**扩展 F-02：** 会签模式 — 全部通过才汇聚

### GA-09 督查督办

**列表字段：** 任务名、责任单位、截止日期、进度%、逾期天数、状态

**逻辑：**

- Timer 事件：截止前 3 天催办（F-06）
- 逾期自动升级至上级部门
- 统计：逾期率 BarChart

### GA-10 政务运行大屏

**布局：** 1920x1080 深色 Schema，fullscreen

**指标：**

- 今日受理量、办结率、平均办结时长
- Map/Bar 各区县办件量（如有地理数据）
- 热点事项 Top5
- 实时滚动办件列表

**刷新：** 60s 自动 refresh（E-09）
